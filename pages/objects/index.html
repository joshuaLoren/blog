Objects

When you are creating objects in Java, you are creating an instance of a class. Classes can extend classes by way of inheritance. In JavaScript objects do not need to be an instance of a class.

We will start start by creating an object in Java. We create objects by creating instances of classes. The Scanner class is build into the Java API and will need to be imported using java.util.Scanner;.

and the desired output:



Lets take a look at the code:


import java.util.Scanner;
Here we are importing the class Scanner, by doing so, we have access to all of Scanner's methods. We will take a closer look at classes later.


Take a close look at line 16: Scanner keyboard = new Scanner(System.in);
Scanner is the class, and we are creating a new instance of this class. The reference variable, "keyboard", is what we will name the instance. By creating an instance of the Scanner class, we have created a Scanner object that will have access to all of Scanner's methods (functions).

This is demonstrated in line 22: name = keyboard.nextline()
Our Scanner object (keyboard), has access to the Scanner class, and uses the method nextLine(). If we were to take a look at the Scanner class we would see a method called nextLine(), that reads the next String input by the user.

Lets take a look at the code:


var computer = {ram : "16 GB", hardDrive : 1 TB 5400 rpm, graphicsCard: "GeForce GTX 1050"}
In JavaScript we have name:value pairs called properties. A JavaScript object is essentially a container for naming properties. One way to access an object property is to use the objectsName.propName. This is just a basic example, but the point of this tutorial is to demonstrate how in a class based language like Java objects inherit methods from classes, while in a nonclass language like JavaScript objects inherit methods (functions) from objects.
